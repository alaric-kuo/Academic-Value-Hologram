## 🛠️ 第四章：技術實作 — 全文拓樸連續積分

### 1. 拒絕剛性分割
傳統分析工具會將文本切割成段落或僅分析摘要，這在 QTE 視角下是「死亡的觀測」。
本引擎採用**重疊滑動視窗積分法 (Topological Overlapping Integration)**：
* 讀取全文 `.md` 或 `.tex` 源碼。
* 以 1500 字元為視窗，800 字元為步長，保留語意的連續演化。
* 將所有局部相位積分為單一宏觀的 **1536 維全局波包 ($\Psi_{global}$)**。

### 2. 三層邊界防禦機制
為防止訊號崩潰（Garbage In），引擎內建防呆邊界：
* **物質邊界**：檢驗有效字元數，阻擋空檔案或純圖片掃描檔。
* **資訊熵濾網**：計算波包與「學術基線向量」的夾角。若相似度過低，判定為邏輯崩潰。
* **算力通道防禦**：對外部 API（OpenAI, Semantic Scholar）設置嚴格超時與中斷機制，不進行推測補齊。

### 3. 語意時間戳 (Semantic Timestamp)
結合 Git 的 Hash 與 QTE 演化狀態。每次上傳文章，系統會自動生成：
* **科學卦詞判定**：定位該文章的演化階段（如：開拓者、奠基者）。
* **PDF 物理投影**：將當下的 Markdown 源碼編譯為 PDF。
* **Github Release**：將當下的狀態永久鎖定在版本紀錄中。
